01 — Overview

Project Summary

This pipeline enriches Vietnam 2024 medical device import trade data by mapping product descriptions against a curated surgical brand model list. Each import row's free-text product description is scanned for known device model or family names; upon a match, the row is tagged with its commercial segment, sub-segment, product classification, manufacturer, and family name — and highlighted in the output file.

03 — Pipeline Step 1

Data Extraction

Because the source Excel workbook contains 520,835 rows across multiple sheets and weighs 132 MB, a streaming extraction strategy was used rather than loading the entire workbook into memory at once.

  • 01
    Streaming read via openpyxl read-only mode

    The RawData sheet was iterated row-by-row using openpyxl's read_only=True mode, which does not load the full DOM into memory — critical for a 520k-row file.

  • 02
    TSV intermediate cache

    All rows were written to a tab-separated flat file (vn_rawdata.tsv, ~175 MB). Subsequent pipeline stages read from this fast-access cache rather than re-opening the Excel workbook.

  • 03
    Keyword reference build

    The V0 reference file's Updated sheet was loaded with pandas. Rows with non-null Model/Family Name values were retained, with pure-numeric entries and strings under 4 characters removed, yielding 3,651 candidate keywords before quality filtering.

04 — Pipeline Step 2

Keyword Matching — Trie Algorithm

Matching 520,835 rows against 3,507 keywords required a sub-linear search strategy. A naïve per-keyword regex scan would have taken over 3,500 seconds; the approach below completed in 7.4 seconds.

Algorithm: 4-character prefix trie

# Build prefix index: first 4 chars → list of keywords prefix_map = defaultdict(list) for kw in sorted(lookup.keys(), key=len, reverse=True): if len(kw) >= 4: prefix_map[kw[:4]].append(kw) def find_match(text, hs4): # HS4 scope gate — skip non-surgical rows entirely if hs4 not in SURGICAL_HS4: return None t = text.lower() for i in range(len(t) - 3): pfx = t[i:i+4] # O(1) hash lookup if pfx in prefix_map: for kw in prefix_map[pfx]: end = i + len(kw) if t[i:end] == kw: # Word boundary check s_ok = (i == 0) or not t[i-1].isalnum() e_ok = (end >= len(t)) or not t[end].isalnum() if s_ok and e_ok: return kw return None

Why this is fast

  • Hash
    O(1) prefix lookup

    At each character position the algorithm performs a single dictionary lookup on the 4-character prefix. Only positions where a known keyword could start are examined further — all others are skipped in constant time.

  • Boundary
    Word-boundary enforcement

    After a candidate keyword is found, its neighbouring characters are checked: the character before the match and after must not be alphanumeric. This prevents "venous" from matching inside "intravenous" and eliminates a large class of false positives without regex overhead.

  • Priority
    Longest-match-first ordering

    Keywords in each prefix bucket are stored longest-first. When multiple keywords share a prefix, the most specific (longest) keyword wins, preventing a short generic keyword from shadowing a longer, more precise one.

⚡
Performance: The trie approach achieves approximately 70,000 rows/second, completing the full 520,835-row dataset in 7.4 seconds — roughly 350× faster than a single compiled-regex alternation over the same keyword set.
05 — Pipeline Step 3

Field Mapping

Upon a successful keyword match, the following fields are populated in the output row from the V0 reference sheet:

V0 Reference Column Output Column Description
C — Segment S — Segment Top-level commercial segment (e.g. "Coronary and Renal Denervation", "Cranial & Spinal Technologies")
D — Sub-segment T — Sub-segment Sub-category within the segment (e.g. "Neurosurgery", "Cardiac Ablation")
E — Product U — Product_V0 Specific product classification from the V0 reference (e.g. "Core Navigation_Capital", "DES")
G — Player P — Manufacturer Manufacturer / brand owner (e.g. "Medtronic", "Abbott", "Boston Scientific")
H — Model/Family Name Q — Family The matched keyword itself — the device model or family name that triggered the match
Derived V — Match_Status "Matched" or "Unmatched" — used as the basis for conditional green highlighting
🟢
Green highlighting is applied via a single Excel conditional formatting rule driven by the Match_Status column (V = "Matched"), rather than per-cell fills. This keeps the file size manageable across 520k rows and ensures the highlighting updates correctly if data is filtered or sorted.
06 — Supplementary Step

Manufacturer Deduction

For matched rows where the V0 reference does not supply a manufacturer (or where the user requested broader coverage), a rule-based deduction step was applied using the Exporter and Importer columns as signals.

  • P1
    V0 Player column (highest confidence)

    Manufacturer taken directly from the matched V0 reference row's Player column. Used wherever a keyword match exists.

  • P2
    Exporter name extraction

    The Exporter field typically names the foreign entity shipping the goods — often the manufacturer's regional trading arm (e.g. "MEDTRONIC INTERNATIONAL TRADING PTE LTD", "CARL ZEISS MEDITEC AG"). A curated rule set of 80+ manufacturer name patterns maps these to canonical names.

  • P3
    Importer name extraction

    When the importer is the manufacturer's own Vietnam subsidiary (e.g. "B BRAUN VIETNAM CO LTD", "JOHNSON&JOHNSON(VIET NAM) CO LTD"), the manufacturer can be deduced. Lower confidence than the exporter signal.

07 — Quality Control

Filters & False-Positive Mitigation

The initial matching pass (using all available keywords against all rows) produced ~258,000 matches — inflated by two classes of false positives that were systematically eliminated in the final version:

Filter 1 — HS4 Scope Gate

Matching is restricted to rows where the HS4 code is within the surgical/medical device universe:

9018 Surgical & medical instruments & appliances 9019 Mechano-therapy / breathing apparatus 9021 Orthopaedic appliances / implants 9022 X-ray / imaging apparatus
⚠️
Why HS4 3006 and 9027 were excluded: HS4 3006 (pharmaceutical preparations including dental materials) was found to produce false matches — keywords like ultimate, nano, and supreme legitimately appear in dental composite brand names. HS4 9027 (analytical instruments) similarly caused false matches with lab reagent products. Both were excluded from scope.

Filter 2 — Keyword Blacklist (77 terms)

Keywords that are common English words, product categories, or material types were removed from the matching set regardless of their presence in the V0 reference file. Key examples:

09 — Limitations

Known Caveats & Recommended Next Steps

Issue Description Severity Recommendation
Homonym keywords Keywords like ranger match both the 3M Ranger warming unit and Bard's Ranger DCB balloon. The correct segment depends on context not captured by keyword alone. Medium Add a secondary HS4 sub-code or manufacturer filter to disambiguate homonyms
Missing HS4 scope Rows where HS4 is blank or outside the 4 surgical codes (9018–9022) are left unmatched even if their description contains a valid device name. Medium Manual review of HS4 null rows; consider extending scope to include 3006 with a tighter keyword whitelist
Generic blacklisted keywords 77 keywords were suppressed to reduce false positives. Some (needle, screw, trocar) are legitimate device names that may result in true matches being missed. Low Reinstate specific multi-word forms (e.g. "Veress needle", "pedicle screw") while keeping single generic words blacklisted
Manufacturer gaps 63.5% of rows (330,605) have no manufacturer populated — primarily rows with unrecognised distributors or non-device imports. Low Provide a distributor-to-manufacturer lookup table for a second-pass enrichment
Single-match-per-row Only the first (longest) matching keyword is recorded per row. In rare cases a product description may contain multiple device names. Low Acceptable for most use cases; extend to multi-match if needed for complex bundle shipments
